В созданном новом запросе через метод GET отправляем запрос по адресу http://162.55.220.72:5005/

162.55.220.72 - это адрес железа
Внутри железа запущен http сервер с операционной системой Linux
У Linux есть возможность работы со множеством портов

На одном железе 162.55.220.72
На разных портах 5005 5007
Могут быть запущены разные http сервера
Т.е. не обязательно на одном ip запускать только лишь один единственный http сервер программу.

http сервер это программа, которая запускается на сервере.
Получается, http сервер 162.55.220.72 слушает 5007 порт.
И когда мы обращаемся в 5007 порт с запросом слэш, у нас возвращается немного другой ответ (по сравнению с портом 5005)

На одном и том же 162.55.220.72 и одном и том же 5005 не может быть запущено два и более http сервера.

Отправляем запрос по GET http://162.55.220.72:5005/get_method
Этот эндпоинт ожидает два параметра на входе, т.е. нам надо чтобы мы отправили два параметра.

Во вкладке Params, в параметре KEY, в строки Key пишем name и age.
Далее в параметре VALUE к name пишем Anna, к age пишем 18.
Сохраняем и Send.

Во вкладке body и далее в подвкладке Pretty должен выйти результат ответа 
[
    "Anna",
    "18"
]

Т.е. на сервере написана логика, что если меня спрашивают что-то методом GET, мне необходимо забрать из URL параметр name и посмотреть какой у него VALUE, и, параметр age и тоже посмотреть какой у него VALUE.
